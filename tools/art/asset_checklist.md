# 原型版 AI 美术资产清单与验收标准(gpt-image-2 全量生成)

风格圣经:`style_bible.json` **v1**(已锁定)。全部 67 条提示词已在 `out/prompts.json` 就绪,按下表 id 取用。

## 通用规则(适用所有条目)

- **提示词**:一律用 `out/prompts.json` 里的成品,不手写。要改风格 → 改 `style_bible.json` → 重跑 `generate_prompts.py`。
- **锚点图**:凡插画类(卡面/领袖/Key Art/结算图),生成时把 `anchors/` 里 3 张 v1 锚点图作为参考图一并给 gpt-image-2;UI 类(框/图标/纹理)不挂锚点,靠提示词里的色板词。
- **负面提示词**:gpt-image-2 不支持独立负面输入,`prompts.json` 的 `negative` 字段作为"验收否决表"使用——出现其中任何元素(文字、水印、卡框、3D 渲染感…)即打回。
- **候选数**:卡面/领袖每张 2~4 候选四选一;UI/棋盘/Key Art 每项 2~3 候选。
- **存放与命名**:`tools/art/generated/v1/<id>.png`,与 prompts.json 的 id 严格一致(已有 3 张锚点卡照此命名)。原始生成图全部保留,裁切/去背产物后续由 postprocess 脚本输出到 `game/assets/`,不手工裁。
- **通用验收**:①并排放进已验收图堆里不跳戏;②冷灰暖褐主调 + 辉尘青金点缀;③画内无任何文字/数字/边框。

## A. 卡面插画 — 49 张(核心工作量)

| 项 | 规格 |
|---|---|
| id | 各卡牌 id(iv_* 16 张、wp_* 16 张、neutral 17 张) |
| 生成尺寸 | **单位卡(36 张):1024×1536 竖版**——立牌取全身,竖版让人物撑满画面,分辨率利用率最高;**指令卡(13 张):1536×1024 横版**——无立牌,动作小景与卡框插画窗比例匹配 |
| 后处理去向 | 卡面从画面上/中部裁 4:3 入框;**单位卡额外去背裁全身立牌**,指令卡不出立牌 |

**单位卡验收**(36 张):单主体全身、含脚、3/4 视角、地面阴影;背景虚化干净(想象去背后轮廓是否完整不粘连);顶左侧轮廓光。**主体不完整/背景杂物粘身 = 打回**,这直接决定立牌质量。另外头顶和脚下各留一点呼吸空间(不顶天立地),卡面 4:3 裁切才有余地。

**指令卡验收**(13 张):单一视觉焦点的动作小景;缩到手牌大小(约 120px 宽)仍能一眼读懂在干什么;边缘暗化虚化。

**阵营辨识**:铁誓=钢灰蓝+誓火金;野群=兽皮褐+野性绿/骨白;中立=青铜/赭黄、**无阵营纹章**。盖住卡名能认出阵营为合格。

## B. 领袖头像 — 2 张

id:`leader_iv_valen`、`leader_wp_saen`。1024×1024(维持正方形——最终是圆形裁切,正方形是圆裁的自然来源),后处理 512 圆形裁切。
**验收**:半身像;脸和纹章在**圆形裁切内**完整可读(生成后拿圆框套一下);气场压得住选人界面。

## E. UI 套件 — 12 项(AI 出元素,排版交给 Godot Theme)

| id | 用途 | 尺寸 | 关键验收点 |
|---|---|---|---|
| `card_back` | 卡背/对手手牌 | 1024×1536 竖 | 严格左右对称;缩小到手牌尺寸不糊成一团 |
| `frame_iron_vow` / `frame_wildpack` / `frame_neutral` | 三阵营卡框 | 1024×1536 竖 | 上 2/3 有**干净的空矩形插画窗**,下 1/3 素面文本区;边框适合九宫格拉伸(四角装饰、四边纹样均匀);三张框窗口位置一致 |
| `gem_cost` / `gem_atk` / `gem_hp` | 费用/攻/血宝石 | 1024×1024 | 去背后缩到 **64px 仍清晰可辨**;三颗风格成套(同挂前两颗已过图作参考再生成第三颗);数字由程序叠加,图内无数字 |
| `emblem_iron_vow` / `emblem_wildpack` | 阵营纹章 | 1024×1024 | 剪影够强,单色化后仍可辨;用于选卡组界面/回合指示 |
| `texture_parchment` | 面板底纹 | 1024×1024 | 低对比、无明显重复痕迹;叠黑色半透明文字可读 |
| `button_plate` | 按钮底板 | 1536×1024 横 | 适合九宫格;素面无字(文字/hover/pressed 状态由 Godot 调色) |

**UI 类通用**:平视、对称、纯色深底方便去背;宁可朴素不可花哨——UI 是 AI 弱项,一旦某项 roll 3 轮仍不合格,**降级为程序绘制**(Godot StyleBoxFlat),不死磕。

## 数量与工时预估

| 类别 | 唯一资产 | 预计生成量(含候选/重roll) |
|---|---|---|
| 卡面 | 49 | ~150-200 张 |
| 领袖+Key Art+棋盘+结算 | 7 | ~25 张 |
| UI 套件 | 12 | ~35 张 |
| **合计** | **68** | **~200-260 张** |

人肉筛选(四选一)按卡面 2~3 秒/张判断 + UI 细看,总投入约 **3~4 小时**,建议分 3 批:①棋盘+UI 套件(先让战场换装)→ ②49 卡面 → ③主菜单+结算。

## 建议生成顺序

1. **C 棋盘 + E 卡框/宝石/卡背** — 最先让 BattleScene 脱掉几何占位,整体观感立刻成型
2. **A 卡面 49 张** — 大头,分阵营批量过(同阵营连续生成一致性更好)
3. **B 领袖 → D Key Art** — Key Art 依赖领袖人设定稿,顺序不能反
4. **F 结算图** — 等 P4 做结算循环时再出
