# 更新日志 · Changelog

《守线 Hold the Line》面向玩家的更新记录：收录**新内容、平衡与规则调整、体验与操作优化、bug 修复**，技术向改动只讲它带来的效果和体验，不写实现细节。按版本倒序排列。

> 标题格式为 `游戏版本 · 日期 · 主题`。游戏版本是面向玩家的**构建版本**（主菜单角标显示、驱动自动更新），按发布递增；涉及规则/卡池的改动会在条目内额外标注对应的**规则版本**，方便开发者对照代码（体验/账号类版本可能只动构建版本、规则版本不变）。

## v0.8.2 · 2026-07-23 · 平衡补丁 #5：围猎叠加、影子炮台完整复制与多阵营数值

> 一批**数值与机制调整**（规则版本 0.10.1 → **0.11.0**）。涉及规则与卡表，**联机需服务端与客户端同步更新到本版本**（旧版本会被版本闸门拦下）；单机 / 人机即时生效。

### 机制
- **围猎现在会叠加**：近战攻击时，**每有一个友方与目标相邻就 +2 伤害**（可叠加），不再是"有人就固定 +2"。把猎物围得越死，咬得越狠——包夹流狂猎显著变强。
- **首席大匠的影子炮台改为完整复制**：召唤出的影子炮台现在**照搬本体的当前状态**——已经掉了多少血、身上有什么增益/减益都一并复制，而**不再附带"突袭"**。等于多一门"当下这门炮"，而不是凭空多一门满血速射炮。
- **自毁保险舱不再占用模块位**：装上保险舱后，炮台仍可另外装满 **5 个升级模块**（保险舱额外挂载，不再挤占升级位）。

### 数值
- **匠会**：分裂弹药 3→**2 费**；移动要塞血量 10→**9**；主炮核心·贯日 +3/+3→**+3/+2**。
- **中立**：御前枪骑加上**疾行 1**，且**牌组上限 1 张**。
- **狂猎**：母狼王血量→**6**；游群之王·卡鲁姆血量→**7**；掠猎巨狼 4/4→**3/5**；掠群幼狼（衍生物）1/1→**1/2**。
- **教团**：灼誓狂徒 3/1→**1/3**；灰烬侍徒 1/1→**1/2**；烬爆蛾、灰烬幼灵各加**疾行 1**；烬蚀之列、焰幕 5→**4 费**。

### 界面
- **新增"疾行"状态角标**：场上随从带疾行 N 时，会像其他状态一样显示一枚「疾N」徽标，一眼看清谁能多走几格。
- 烬火唱徒 / 烬眼先知 / 灰烬侍徒 的卡面补上「(每回合最多 2 次)」——这条上限本就一直生效，只是此前没写在卡面上。

## v0.8.1 · 2026-07-23 · 排位分改版与烬火陷阱修复

> 一处**排位积分算法改版** + 两处 **bug 修复**（规则版本 0.10.0 → **0.10.1**）。烬火陷阱修复涉及规则，**联机需服务端与客户端同步更新到本版本**（旧客户端会被版本闸门拦下）；单机 / 人机即时生效。排位改动为**纯服务端**，重部署服务端后在线上生效，客户端无需为它更新。卡组联机同步修复为**纯客户端**，随客户端更新即生效，**无需重部署服务端**。

### 排位：赢必涨，被低估会自动爬回
- **赢一局必定涨分**：告别"赢了却不涨分"——每胜一场至少 +5，通常 +20 上下；输一场最多扣 40，不会无限下坠、也不会跌破 0。
- **被低估会加速回正**：当你的真实水平高于当前排位分（例如开局连败沉到底、或久未上分），系统会让你**赢得更多、扣得更少**，几场就爬回应有的分段——不再"明明常赢却卡在底部"。
- **匹配更公平**：对手配对改用一套**隐藏实力分**（只用于匹配、不显示），你看到的排位分则专注反映"战绩进度"。两者分工后，排位分涨跌更符合直觉，配对也更贴近真实水平。

### 修复
- **烬火陷阱不再同回合双重灼蚀**：此前在**自己回合**踩上烬火陷阱，会在**踩上瞬间**与**同一回合结束时**各灼蚀一次，单回合白白承受双倍伤害。现在踩上只吃**进入时**那一下，随后安全度过自己回合末与敌方回合，直到**你的下一个回合结束**才会再次灼蚀，火焰约烧两回合后熄灭。无人占据的火焰仍会正常倒计时熄灭，不会永驻。
- **删掉 / 失效的卡组不再残留在联机大厅**：此前在卡组管理里删掉一套卡组（尤其是**离线删除**时），或某套卡组因版本更新引用了已被移除的卡牌，它仍会出现在联机对战的卡组选择里。现在——删除会在下次联网时**可靠地同步到服务器**，不再"删了又回来"；**失效**（含已移除卡）的卡组会在大厅标出 **⚠** 并禁止用它排位，并给出修复 / 删除指引；同时新增 **「清理失效卡组」** 入口，一键清掉那些只存在于服务器、本地管理器够不到的残留卡组。你在别的设备 / 账号上建的卡组不受影响。

## v0.8.0 · 2026-07-22 · 掘世匠会重做：单核炮台与模块升级

> 掘世匠会（械）整阵营重做——从「一堆同质炮塔随从」变成「全阵营只造一门炮，牌库是零件箱、对局是装配台」。这是一次涉及**规则、卡池、领袖技能**的大改（规则版本升至 **0.10.0**）。**须同步重部署服务端并更新客户端**：卡表数据变化后，未更新的旧版本无法与新版本联机（会被版本闸门拦下）。单机 / 人机即时体验。

### 新玩法：一门炮，你来组装
- **领袖「铸炮」**：总工·布罗姆的技能改为 **1 费在底线放下一座 1/3 射程 2 的工造炮台**（每方场上至多一座，已有炮台时按钮置灰）。裸炮可移动、可挨打，但脆——开局要护住它。
- **模块升级（全新卡类）**：14 张**模块卡**打出即装到你的炮台上，把它拼成你要的样子——狙击塔、AOE 控场塔、履带突击塔、锚定要塞。**装配上限 5**，同名唯一；满位时打新件需选一件在装模块报废（报废件仍留在历史池，可回收）。
- **分级模块取最高**：溅射Ⅰ/Ⅱ、吸血Ⅰ/Ⅱ 同系两档同时在装时，每次结算只按最高的一件计。**吸血**为永久增益——攻击造成伤害后炮台直接 **+0/+N 生命**（吸血Ⅰ +1、吸血Ⅱ +攻击力一半），**可超过原上限**、卸件也不掉。
- **锚定还是机动**：架设平台让炮台 +1/+3、坚守但钉死后场；履带底盘让它 +1 移速自己压线。这是匠会每局的中轴选择。
- **止损与爆发**：自毁保险舱在炮台被击毁时保住 2 件模块给下一座炮台；战地重构从历史池随机装回 2 件；镜像工坊复制一件在装模块（无视同名唯一，开关类复制会提示无增益）；**首席大匠·维尔达**（9 费）战吼在身边召唤一座复制你炮台当前面板与模块的**影子炮台**（突袭，长期存在、直到被消灭）——等于多开一门炮。

### 界面
- **炮台装配面板**：点选炮台，详情面板底部列出它的在装模块与 5 格进度、以及历史池；影子炮台以半透明暗蓝呈现，一眼区分。
- **装配指引与分因提示**：打模块卡时自动装配或弹出报废/复制选择弹层；打不出时按原因提示——「需要炮台在场」/「已装备相同模块，不可装备重复模块」/「没有可复制的模块」。
- **加属性 buff 飘字**：所有增加攻/血的强化（齿轮工长、护炮班组、吸血、驻防等，全阵营通用）现在会飘出 `+攻/+血` 数值，和治疗的飘字对齐。

### 卡组迁移
- 掘世匠会砍掉了一批同质旧卡。**若你的自建卡组含已移除的卡牌**，进入对战时会明确提示「卡组含 N 张已移除卡牌，请在编辑器中修复」，而不是报错——不会自动替换，构筑决定权仍在你手里。内置「铸炮阵列」预组已更新。

## v0.7.7 · 2026-07-22 · 稳定性与幕后架构优化

> 一批面向稳定性的改进与大规模幕后重构：联机重连 / 匹配更可靠、本地存档更防损坏，以及规则引擎与对战界面的架构整理。**规则与卡池零改动**（规则版本仍为 **0.9.1**，与 0.7.6 及更早版本联机兼容），对局玩法、数值与结算结果完全不变。联机侧的稳定性改进需**重部署服务端**后才在线上生效；单机 / 人机即时生效。

### 稳定性
- **联机重连与匹配更可靠**：修复了重连握手在若干边界情况下的处理（失效的重连请求会被明确终结、错误回执按对应请求正确关联），排队中掉线会自动重新入队，对战数据管线也做了加固，减少长局或网络抖动时的异常。
- **本地存档更防损坏**：账号、设置、牌组等本地文件改为原子写入（先写临时文件再整体替换），中途断电 / 崩溃不会写坏原文件；若检测到已损坏的旧文件，会自动隔离留档并回退到可用状态。
- **数据缺失时给出明确提示**：卡表等核心数据文件缺失或读取失败时会弹出清晰的自检提示，而不是静默出错。

### 幕后
- **规则引擎与对战界面架构重构**：对规则引擎和对战场景做了大规模内部整理（卡牌效果改为统一动作注册表、伤害减免计算收敛为单一路径、对战界面拆分为多个可独立测试的模块并改用增量渲染）。这一切**不改变任何玩法、数值或结算结果**——439 项测试与对局模拟逐字节一致——目的是为后续新阵营与新机制的扩展打好地基。

- **选目标提示分色**：需要选择时，提示栏会用加粗红色标出「作用目标」、蓝色标出「引导者」、绿色标出「接受目标」，棋盘单位高亮同步使用对应颜色。
- **无目标引导牌可以正常选人**：燔火、燎原等不需要指定伤害目标的引导牌，在场上有多个友方随从时会正确要求选择引导者，不再卡在错误的目标提示上。
- **同屏对战双方都能选牌组**：玩家一和玩家二可分别选择内置或本地牌组，游戏会记住双方上次选择；回合交接提示也会按实际领袖阵营显示。

### 电脑对手
- **AI 会选择更合适的指令卡释放引导者**：电脑现在会比较不同引导者带来的薪炎增伤、击杀收益与费用减免，不再在效果不同的引导方案之间近似随机选择。

### 修复
- **定身状态现在会正确显示**：被「灰缚」等定身后，随从卡面右侧的减益指示器会亮起「定身」标记，点开详情面板也会列出定身说明。此前因为定身是"限时状态"，客户端一直没把它显示出来（徽标与描述都缺失，尽管规则上确实无法移动），现已修复。

### 幕后
- **状态指示器改为数据表管理**：随从身上的状态徽标（持盾 / 福泽 / 定身 / 熔岩巨剑…）改由可在编辑器内直接增删管理的资源表驱动，方便后续快速新增状态。玩家看到的显示与此前一致。

## v0.7.4 · 2026-07-21 · 教团法术改版与秘密体系（平衡补丁 #4）

> 本次是**教团阵营的大改版 + 一批全新机制**（规则版本 0.8.1 → **0.9.0**），联机需服务端与客户端**同步更新到本版本**后才能对局；单机 / 人机不受影响。完整数值见 `balance/BALANCE-CHANGELOG.md` 补丁 #4。此前待发布的「UI 全面美化改版」也随本版一并上线（见文末）。

### 教团法术改版（核心）
- **直伤要选位了**：教团的点伤不再"隔空全场点名"。战吼类改为**锚·N**（以随从落地格为中心，曼哈顿 N 格内选目标），指令类改为**引导·N**（施放前先选一个友方随从当"引导者"，目标须在引导者 N 格内）。
- **引导者差异化**：由谁引导会改变效果——**焰术学徒**引导 → 该薪炎指令伤害 +1；**熔岩巨灵**引导 → +2；**晚祷领唱**引导 → 该薪炎指令费用 -1（下限 1 费）。这三张随从的旧战吼/触发已换成上述"引导加成"。
- **薪炎伤害 + 加深 / 蓄能**：教团的伤害与灼蚀统一为**薪炎**类型。**焰跃术士**战吼改为**蓄能 2**（你的下一张薪炎伤害指令 +2）。
- **归魂**：**灼誓狂徒**新增——你的回合内每有友方随从死亡，获得 1 点本回合辉尘（每回合最多 2 次）。
