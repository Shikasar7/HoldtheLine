## 0. 设计总纲

GDD §5 给了两个阵营的空间人格,本稿把它们落到可施工粒度:

| | 黄昏教团(Duskweavers)"焰" | 掘世匠会(Undervault)"械" |
|---|---|---|
| 空间人格 | **以格/行/列为目标的法术**;身材差、指令强;误伤友方是代价也是燃料 | **阵型即答案**;射程密度最高;架设炮台不能动,前排护后排 |
| 玩法原型 | 法术连锁/节奏:场面可以输,一套烧穿一列 | 后排火力/组合成型:摆好阵地,隔墙点名 |
| 阵营引擎 | 新触发器 **`ally_order_played`**(你每施放一张指令牌后触发单位效果)——单位是"法术放大器" | 新关键词 **架设**(不能移动,身材折价)+ **贯穿**(远程穿透补伤)——位置一旦选定就是承诺 |
| 与压力潮汐的张力 | 法术不占格,推线必须靠脆皮单位——潮汐天然逼教团上人 | 架设单位永远进不了敌方半场——匠会必须留机动分队推线 |
| 输给谁(软克制) | 风(游群):快攻贴脸,法师来不及连锁 | 焰(教团):行/列 AOE 洗静态阵型 |
| 赢谁(软克制) | 械(匠会):AOE 点名炮阵 | 墙(铁誓):射程隔墙点名,墙堵不住炮弹 |

> 设计纪律复述(GDD §4):**任何新关键词必须与"格子"发生关系**。本稿新增的 5 个原语(§6)全部以格/行/列/相邻为作用域。

### 新增数值税(接 docs/04 §1 公式:价值点 = 1.3×攻 + 1×生命 + 税;预算 ≈ 2.2×费 + 1.2)

| 新条目 | 税 | 说明 |
|---|---|---|
| 射程 1 | 1.2 | 贴脸免反击炮;首见于短管铳手 |
| 架设(不能移动) | **-1.0** | 负税=身材补偿;同时永远吃不到潮汐豁免、不能回防换线 |
| 贯穿 | 1.5 | 仅远程攻击生效;对目标正后方一格的单位补同额伤害 |

| 消灭(destroy,指向友方) | 作为成本计 **-2** | 献祭系:亏一个身位换牌/费 |

## 3. 阵营机制细则(结算语义,施工前必读)

### 3.1 ally_order_played(教团触发器)

- 时机:**你**打出一张指令牌、其 play 效果全部结算完毕之后;按单位入场顺序依次触发(与亡语 sweep 同序,保证重放确定性)。
- 只认 `CardType.Order`(军令硬币也算!"硬币触发全场引擎"是教团的爽点 combo,保留);领袖技能**不算**。
- 对手施放指令**不触发**(所以叫 ally_order_played)。
- 触发效果的目标选择器限定为无需显式目标的类型:`self` / `adjacent_enemies` / `adjacent_allies`(打一张牌不该弹出二次选目标)。CardDatabase 载入时校验。

### 3.2 架设(Emplacement)

- 不能移动:MoveUnitCommand 直接拒绝;跃障、move_bonus 均无效(没有移动力可加)。
- **受到效果伤害 +1**(指令/领袖技/战吼亡语等一切非攻击伤害;v2.1 平衡定稿新增)——钉死的炮躲不开落下的弹幕。这是"焰克械"的机制接口:全游戏的指令伤害几乎都在教团手里,狼群零指令伤害不受影响。攻击伤害不加成。
- **可以**被召唤/部署到任何合法格(仍只有己方底线行)。
- 与坚守天然联动(永远没动过);与压力潮汐天然互斥(永远无法进入敌方半场)——匠会的推线税。
- UI 徽记:「架」;立牌可加底座钉表现。

### 3.3 贯穿(Pierce)

- 仅**远程攻击**(range>0 路径)生效:结算对目标的伤害后,沿攻击直线找目标**正后方一格**(远离攻击者方向),若有单位(**不分敌我**)则受到等额伤害。
- 只穿 1 格、只穿 1 次;后方单位的持盾/坚守正常生效;贯穿伤害不触发反击。
- "不分敌我"是刻意的:炮线打自己人是匠会的站位考题,与教团的友伤呼应。
- UI 徽记:「贯」。

### 3.4 destroy / target_unit_ally

- destroy:直接置 0 生命进死亡 sweep(亡语照常),不走 DamageUnit(持盾/坚守不挡献祭)。
- target_unit_ally:显式目标 + 载入期/结算期校验友方(现有 target_unit 不校验敌我,保持不动,避免波及旧卡)。

## 5. 平衡验证计划

1. 施工完成后 Sim 扩成 **4 阵营循环赛**(6 个配对 × 轮换先手 × ≥1000 局/配对,greedy 口径):目标任意配对 40–60%,克制边允许压到 40–45%。
2. GreedyAi 需要学会的新事:架设单位移动打分跳过;贯穿的直线补伤计入攻击评分;cell_cross_all 友伤扣分;destroy 只对"将死/低值+手牌收益"打正分;on-cast 引擎在场时指令牌加分。
3. 压力潮汐观察项:两个新阵营都天然不爱推线(法术不占格/炮台不能走),若潮汐致死率异常升高,优先调 PressureTideStartRound=10 而不是改阵营卡。
4. 首张改卡候选(按风险排序):门德的双触发、贯日 5/8 身材、献祭抽 2、灼痕友伤半径。

## 8. v2.1 平衡定稿(2026-07-17,Fable)

### 8.1 裁决记录

**① 熔岩巨灵战吼目标(正式裁决,非权宜)**:采用 `unit_cross_all` 选择器——以**选中的一个随从(可为友方)**为中心画十字,复用 TargetUnitId,卡面文案「对一个随从及其相邻格的所有随从造成 1 点伤害(含友方)」。给 PlayCardCommand 加第二个 cell 字段的方案**否决**:部署格 5 × AOE 中心 20 = 单卡 100 个候选命令,枚举器"生成后逐个 dry-run"的架构下 AI 与合法性成本十倍;且为一张卡永久扩宽命令/日志协议面不值。表达力损失很小:以友方为爆心恰是教团友伤流的爽点,瞄空格的幻想由晚祷烈焰(指令,cell 目标)承载;战吼选随从、指令选格子,两套瞄准范式各自纯粹。**待地形格落地(格目标成为系统能力)时重议**。

**② 架设第二条语义**:受到效果伤害 +1(见 §3.2)。教团 vs 匠会曾钉死在 33-35%——诊断(单配对 66.8% 潮汐终结)显示这是放血竞赛:教团 2 伤 AOE 打不死 4-6 血炮台,脆皮引擎被远程无反击点名,推不进半场持续流血。数值补丁(拆炮工具 +1 伤)两轮无效,关键词级反制一步到位(34→46-52)。

**③ GreedyAi 潮汐意识**(X2.1 补漏):回合数临近/进入潮汐期时,"成为己方在敌方半场唯一单位"的移动获得递增加分(封顶 10),撤出反向扣分。没有它,bot 用放血竞赛代替战斗,矩阵测的不是卡牌强度。

### 8.4 火种循环(2026-07-17,回应"指令太少且无法循环",数值未定稿)

新原语 **`recall_order`**(EffectSpec action):将 N 张**随机**指令牌从你的墓地移回手牌。随机取样走对局 Rng(重放确定);墓地中的单位永不可选;复用抽牌管线的手牌上限语义(满手爆牌)与 CardDrawn/CardBurned 事件——**零新事件、零协议变更**。GreedyAi 按"墓地里实际有几张指令"折算抽牌分。

挂载(数据):
- **焰跃术士**(稀有 4 费 3/3):战吼由"1 伤"改为**回收 1 张**,保留突袭——回防+续弹双职责;
- **灰烬凤凰**(史诗 6 费 5/6):亡语在召唤雏凤之外**再回收 1 张**——凤凰=复燃,主题闭环;
- 预组「晚祷」:烬爆蛾 3→2、焰鞭 2→3(指令 9→10)。

⚠ **平衡未定稿**(用户要求先实装试玩):首测 greedy 矩阵显示这版回收对教团是全面大 buff(vs 狂猎冲到 71%)。试玩后若同感,预备的下调选项按序:焰跃术士 3/3→2/3 → 去突袭 → 预组焰鞭回 ×2 → 凤凰去回收(只留焰跃术士单点)。
