# 《守线》联机部署与测试指南(M2)

> 面向 M2 的验收:两台电脑隔网打完整局、拔网线能重连。本指南给三条由易到难的联网路径,
> 以及自测清单。服务器与客户端都已就绪(N0–N3),这里只讲怎么把它们跑起来、连起来。

---

## 0. 组件速览

- **对战服务器**:`src/HoldTheLine.Server`,ASP.NET Core + WebSocket,监听 `0.0.0.0:5210`(可配)。
  - 健康检查:`GET /healthz` → `ok`。对战端点:`GET /ws`。
- **游戏客户端**:Godot 导出的 `HoldTheLine.exe`,主菜单「联机对战」填服务器地址即可。
- **陪练机器人**(可选,自测用):`src/HoldTheLine.BotClient`,能加入房间随机出牌打完一局。

配置项(环境变量 `HTL_` 前缀,或 `appsettings.json` 的 `Server` 段):

| 键 | 默认 | 说明 |
|---|---|---|
| `Server__Urls` | `http://0.0.0.0:5210` | 监听地址 |
| `Server__TurnSeconds` | `90` | 回合时限,超时服务器代打 EndTurn;同一玩家连续两回合超时判负 |
| `Server__DisconnectGraceSeconds` | `120` | 掉线宽限,窗口内用 resume_token 重连可续打;超时判掉线方负 |
| `Server__CommandLogDir` | 空(关闭) | 设为目录则每局落一份 `match-*.jsonl`(配置头 + 每条命令),可重放排查 |

## 1. 本机自测(最快,先跑通)

一台电脑上开服务器 + 一个 Godot 客户端 + 一个机器人陪练:

```bash
# 1) 起服务器
dotnet run --project src/HoldTheLine.Server -c Release
# 看到 "listening on http://0.0.0.0:5210" 即就绪;另开一个终端验证:
curl http://127.0.0.1:5210/healthz    # → ok

# 2) Godot 客户端:主菜单 → 联机对战 → 服务器填 ws://127.0.0.1:5210/ws → 创建房间 → 记下房间号

# 3) 机器人加入该房间陪你打(把 ABC123 换成上一步的房间号)
dotnet run --project src/HoldTheLine.BotClient -c Release -- --join ABC123 --play --url ws://127.0.0.1:5210/ws
```

千局压力自测(纯机器人对打,验证不崩不卡):

```bash
dotnet run --project src/HoldTheLine.BotClient -c Release -- --selfplay 1000 --url ws://127.0.0.1:5210/ws
```

## 2. 局域网(两台同 WiFi/网线)

服务器机查本机内网 IP(如 `192.168.1.20`),放行 5210 端口的入站:

```powershell
# 服务器机(Windows,管理员 PowerShell)一次性放行:
New-NetFirewallRule -DisplayName "HoldTheLine 5210" -Direction Inbound -Protocol TCP -LocalPort 5210 -Action Allow
```

- 服务器机:`dotnet run --project src/HoldTheLine.Server -c Release`(默认已监听 0.0.0.0)。
- 两台客户端主菜单「联机对战」服务器都填 `ws://192.168.1.20:5210/ws`,一人创建、一人用房间号加入。

## 3. 隔互联网:Tailscale 组网(推荐,零公网、零端口映射)

对 3~5 人测试最省事:你和朋友都装 [Tailscale](https://tailscale.com),登录同一账号(或用共享),
双方设备就进了同一个虚拟内网,彼此有稳定的 `100.x.y.z` 地址,不用买服务器、不用端口映射。

1. 双方安装并登录 Tailscale。服务器机跑 `dotnet run --project src/HoldTheLine.Server -c Release`。
2. 服务器机执行 `tailscale ip -4` 拿到它的 Tailscale IP(形如 `100.101.102.103`)。
3. 两台客户端「联机对战」服务器填 `ws://100.101.102.103:5210/ws`,创建/加入即可。
   - Windows 防火墙可能仍需放行 5210(见 §2 的命令);Tailscale 接口通常被视为专用网络。

> 这条路径即 M2 计划 §9 的默认方案(待定夺 D-M2-1)。打完 DoD 后若要长期在线,再考虑 §4 的 VPS。

## 4. 公网 VPS(长期在线,可选)

一台入门级云主机即可(回合制并发压力极小):

```bash
# 在 VPS 上
dotnet publish src/HoldTheLine.Server -c Release -o /opt/htl-server
# 需要 game/data 与可执行文件在一起(服务器自动向上查找 game/data);把仓库的 game/data 一并部署
HTL_Server__Urls="http://0.0.0.0:5210" /opt/htl-server/HoldTheLine.Server
# 生产建议用 systemd 常驻;加 wss(反代 + 证书)是可选加固,非验收项
```

客户端服务器填 `ws://<VPS公网IP>:5210/ws`。云厂商安全组放行 5210。

> M2 不需要 Postgres/Redis(无账号无收藏);命令日志落本地 JSONL 即可。

## 5. 自测清单(对齐 M2 DoD,docs/05 §12)

- [ ] **异地两台电脑经组网/公网打完 ≥3 整局**,全程无崩溃、双方结算一致 ← 核心 DoD
- [ ] **拔网线重连**:对局中把一方网络断开约 30 秒再恢复,客户端显示「重连中…」,恢复后无损续打
      (超过 `DisconnectGraceSeconds` 才判负)
- [ ] **对手掉线提示**:一方掉线时另一方看到「对手掉线,等待重连…」
- [ ] **回合计时**:界面显示 `⏱ 你/对手 Ns` 倒计时;放着不操作会被服务器代结束回合,连续两回合不动判负
- [ ] **认输**:对局中按 Esc 认输,双方进入结算
- [ ] 席位 1(加入方)视角镜像正确:己方底行在屏幕下方

> 自动化护城河已覆盖:两 bot 联网整局 + 裁剪断言 + 命令日志重放一致、断线重连、宽限判负、
> 超时判负、命令日志落盘重放(见 `HoldTheLine.Server.Tests`,14 条)。本清单是人工现场验收补充。
